Here's the full technical breakdown of how ledger PDFs are generated for buyers and farmers:
Library Used

jsPDF — a pure client-side JavaScript library. The PDF is generated entirely in the browser, no server-side rendering needed.
Page Layout

    Format: A4 portrait (210mm x 297mm)
    Margins: 12mm on all sides
    Content width: 186mm (210 - 12×2)
    Units: Millimeters throughout

Structure (top to bottom)

1. Business Header

    Business name — centered, bold, 13pt Helvetica
    Address line — centered, normal, 8pt

2. Report Title

    "Buyer/Farmer Ledger: [Name] ([ID])" — bold, 10pt
    FY date range (e.g., "01 Apr 2025 to 31 Mar 2026") — normal, 8pt

3. Table Header

    A filled green rectangle (RGB 46, 125, 50) drawn as background
    White text for column labels: #, Date, Particulars, Dr, Cr, Balance
    Header height: 7mm
    This same header is redrawn on every new page

4. Column Positions (X offsets from left margin)
Column	X Position	Alignment
# (index)	margin (12mm)	Left
Date	margin + 10 (22mm)	Left
Particulars	margin + 30 (42mm)	Left
Dr	margin + 127 (139mm)	Right
Cr	margin + 150 (162mm)	Right
Balance	pageWidth - margin (198mm)	Right

5. Data Rows

    Row height: 6.5mm
    Zebra striping: Even rows get a light grey background (#F8F8F8)
    Opening/Closing Balance rows: Light green background (#E8F5E9), bold text
    Particulars truncation: Strings longer than 45 characters are cut with "..."
    Currency: Indian locale formatting (en-IN), no decimals
    Font: Helvetica, 7.5pt for data

6. Footer

    "Generated by Mandi Mitra" watermark at bottom center, 7pt

Pagination Logic

    Tracks current Y position as rows are drawn
    When Y exceeds 275mm (near page bottom), calls doc.addPage()
    Resets Y to top (14mm) and re-draws the table header on the new page

Balance Calculation (Key Difference)
	Buyer Ledger	Farmer Ledger
Running balance	balance + dr - cr	balance - dr + cr
Positive display	"Amount Dr"	"Amount Cr"
Negative display	"(Amount) Cr"	"(Amount) Dr"
Balance coloring	Uniform black	Red for Dr, Green for Cr

This is because for buyers, Dr = receivable (what they owe), Cr = payment received. For farmers it's the opposite direction.
FY Filter Flow

    User selects FY from a dropdown (current + 4 previous years)
    API call to /api/buyers/:id/ledger?fy=2025 (or farmers equivalent)
    Server calculates the opening balance by summing all pre-FY transactions:
        Opening Balance = static openingBalance + sum(pre-FY Dr) - sum(pre-FY Cr)
    Returns: openingBalance, fyStart, fyEnd, and entries[] for that FY window
    Client builds the row array: Opening Balance row → sorted entries → Closing Balance row

File Output

Saved as: Buyer_Ledger_[Name]_FY[Year].pdf or Farmer_Ledger_[Name]_FY[Year].pdf
